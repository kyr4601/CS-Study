# 네트워크
- 네트워크란?
  
  - 컴퓨터나 기타 기기들이 리소스를 공유하거나 데이터를 주고 받기 위해 유선 및 무선으로 연결된 통신체계

  - 네트워크는 통신하기 위해 존재
   
  - 따라서 약속된 통신 방법 필요 (두 사람이 대화하려면 같은 언어를 써야하듯)  
   
* 프로토콜 : 통신 주체들이 참여하는 형식, 절차, 규약

   <img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FHumFD%2FbtqwmCCld9L%2Fh6wCy1lK1eU8UCb6hYvbr1%2Fimg.png" width="700" height="300">
   
- 프로토콜의 역할

  1. 애플리케이션 목적에 맞는 통신 방법을 제공
   
  2. 신뢰할 수 있는 데이터 전송방법 제공
   
  3. 네트워크 간 최적 통신 경로 설정
   
  
- 네트워크 관련 용어
   
    - IP 주소(IP Address): 인터넷 프로토콜 주소로, 네트워크에 연결된 각 장치를 고유하게 식별하는 데 사용되는 번호
   
    - MAC 주소(MAC Address): 네트워크 카드 또는 네트워크 인터페이스 카드(NIC)에 할당된 고유한 식별자
   
    - 라우터(Router): 네트워크 간의 패킷을 전송하는 장치로, 가장 효과적인 경로를 결정해 데이터를 전달
     
    - 스위치(Switch): 네트워크 내에서 데이터 패킷을 특정 목적지로 전송하는 장치
     
    - 서버(Server): 네트워크에 연결된 컴퓨터들에게 데이터나 서비스를 제공하는 컴퓨터 또는 시스템
 
    - 클라이언트(Client): 서버에게 서비스를 요청하는 컴퓨터 또는 시스템
     
    - 프로토콜(Protocol): 컴퓨터나 네트워크 장치가 서로 통신하기 위한 표준 또는 규칙 ex) HTTP, TCP/IP
     
    - 인터넷(Internet): 전 세계의 컴퓨터 네트워크들이 연결된 전세계적 네트워크를 의미
 
    - LAN(Local Area Network): 한정된 지역 내에서 컴퓨터와 다른 장치들이 연결된 네트워크
     
    - WAN(Wide Area Network): 넓은 지역을 커버하는 네트워크로, LAN보다 큰 범위를 커버합니다. ex) 인터넷
     
    - Wi-Fi: 무선 네트워킹 기술로, 컴퓨터, 스마트폰 등의 장치를 인터넷에 연결
     
    - VPN(Virtual Private Network): 인터넷을 통해 가상의 네트워크 연결을 생성하는 기술로, 데이터 전송의 보안을 강화
  
## 네트워크 구조

### 네트워크 엣지
<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2F8ggWx%2FbtqwpTJFxZb%2F0ytsd0y1eRqjN3PG9ExciK%2Fimg.png" width="700" height="400">

- 네트워크 엣지 - Mobile Network, Home Network, Institutional Network : 사용자들이 직접 접하는 네트워크
  
-  네트워크의 '주변'이나 '말단'을 의미하며, 사용자의 디바이스(예: 컴퓨터, 스마트폰 등)나 사용자에게 서비스를 제공하는 서버 등이 위치하는 곳을 가리킨다.

- 엔드 호스트(End Host):  데이터가 전송되는 과정에서 최종적으로 도달하거나, 데이터 전송을 시작하는 장치를 의미합니다.
   
### 네트워크 코어

네트워크 코어 - Global ISP, Regional ISP : 개인/기업에게 인터넷 접속 서비스를 제공하는 회사들이다. 이를 통해 네트워크 엣지들이 연결된다. 

ISP : 개인이나 기업체에게 인터넷 접속 서비스, 웹사이트 구축 및 웹호스팅 서비스 등을 제공하는 회사를 말한다. 대표적으로 한국에서는 KT 인터넷, SK 브로드밴드, LG U+ 등이 있다. 

- 네트워크의 중심을 의미하며, 주로 고성능 라우터들로 구성되어 있다. 라우터는 데이터를 빠르게 전송하고, 다양한 경로 중 최적의 경로를 선택하는 역할을 한다.

### 엑세스 네트워크

<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FXqqdQ%2Fbtqwq6290rD%2FnH9iyWBHGbFlWiyoy3PxC1%2Fimg.png" width="700" height="400">

사용자의 디바이스가 인터넷에 접속하기 위한 첫 번째 접속점, 즉 '엣지 라우터'에 연결하는 네트워크를 의미. 

- Residential Access Network: 주로 가정에서 사용하는 홈 네트워크를 의미한다. 사용자의 개인 디바이스가 집 안의 인터넷에 접속하기 위한 네트워크이다.
  
- Institutional Access Network: 학교나 회사 등의 기관에서 사용하는 네트워크를 의미한다. 이 네트워크를 통해 기관 내의 다양한 디바이스들이 인터넷에 접속한다.
  
- Wireless Access Network: 무선으로 인터넷에 접속하는 네트워크를 의미한다. 휴대폰 같은 모바일 디바이스가 셀룰러 네트워크나 와이파이를 통해 인터넷에 접속하는 경우를 포함한다.

## 네트워크 토폴로지 종류

### 스타 토폴로지
<img src="https://images.edrawsoft.com/kr/articles/edrawmax/topology/star-topology.png" width="700" height="300">

- 모든 노드가 중앙 허브와 직접 연결된 네트워크 구조 
- 중앙 허브가 데이터의 흐름을 관리하며, 각 노드에서 보낸 정보는 반드시 중앙 노드를 지나 목적지에 도달한다.
- 가장 흔한 토폴로지 구조

#### 장점

1. 한 장소에서 전체 네트워크를 관리한다.
2. 각 노드가 중앙 허브와 독립적으로 연결되어 있어, 한 노드에 문제가 생겨도 다른 노드에는 영향이 없다.
3. 전체 네트워크를 오프라인 상태로 만들지 않고도 장치를 추가, 제거, 수정 가능하다.
4. 네트워크 전체를 연결하는데 필요한 케이블이 적어, 네트워크 확장이나 축소가 용이하다.
5. 네트워크 디자인이 단순해 문제 발생 시 쉽게 찾아낼 수 있다.
   
#### 단점

1. 중앙 허브에 문제가 발생하면 전체 네트워크에 영향을 미친다.
2. 네트워크의 성능이 중앙 노드의 설정과 기술 사양에 의해 제한되기 때문에 설정비와 운영비가 상승할 수 있다.

### 버스 토폴로지
<img src="https://images.edrawsoft.com/kr/articles/edrawmax/topology/bus-topology.png" width="700" height="300">

-  네트워크상의 모든 장치가 하나의 케이블로 연결된 구조를 말한다. 
- '선형 토폴로지'나 '백본 토폴로지'라고도 불린다.

#### 장점

1. 비용 효율적이고 소규모 네트워크에 적합하다.
2. 배치가 단순하고 추가적인 노드 연결이 쉽다.
   
#### 단점

1. 하나의 케이블만을 이용해 데이터를 전송하기 때문에 케이블에 문제가 생기면 전체 네트워크에 영향을 줄 수 있다.
2. 노드를 추가할 때마다 대역폭이 낭비되어 데이터 전송 속도가 느려질 수 있다.
3. 데이터를 동시에 양방향으로 보낼 수 없어 많은 트래픽을 전송해야 하는 네트워크에는 부적합하다.
  
### 링 토폴로지
<img src="https://images.edrawsoft.com/kr/articles/edrawmax/topology/ring-topology.jpg" width="700" height="300">

-  네트워크상의 모든 장치가 원 또는 고리 형태로 연결된 구조를 말한다. 
-  각 장치는 양쪽 이웃한 장치에만 연결되어 있고, 데이터는 원을 따라 한 방향 또는 양방향으로 흐른다.

#### 장점

1. 데이터 패킷이 충돌 없이 효율적으로 전송될 수 있으며, 비용 효율적이고, 문제를 쉽게 찾을 수 있다는 점이 있다. 
2. 대규모 네트워크에서는 데이터 손실 없이 패킷을 목적지에 도달시킬 수 있도록 리피터를 사용할 수 있다.
   
#### 단점

1. 노드 하나에 문제가 생기면 전체 네트워크가 중단될 수 있으며, 네트워크를 안정적으로 관리하기 위해 노드를 주시해야 한다. 
2. 네트워크에 장치를 추가하면 통신 지연이 발생할 수 있으며, 노드를 재설정하거나 추가 또는 제거하기 위해서는 전체 네트워크를 중단해야 하는 불편함이 있다.

### 트리 토폴로지
<img src="https://images.edrawsoft.com/kr/articles/edrawmax/topology/treetopology.png" width="700" height="300">

-  네트워크 구조에서 나무와 같이 중앙 노드에서 뻗어나가는 구조를 가진 토폴로지를 말한다. 
-  이 구조는 스타 토폴로지와 버스 토폴로지의 요소를 결합한 것으로, 부모-자식 계층 구조로 노드들이 연결되어 있다.

#### 장점

1. 네트워크 확장이 쉽고, 각 가지에 개별적으로 접근이 가능해 네트워크 장애 해결이 비교적 간단하다.
   
#### 단점

1. 전체 네트워크의 안정성이 중앙 노드에 의존하며, 중앙 노드에 문제가 발생하면 가지들 사이의 연결이 끊어질 수 있다.
2. 노드를 추가할수록 네트워크 관리가 어려워지고 비용이 증가하며, 특히 각 장치를 연결하는 케이블의 양이 많아 설치 비용이 높다.
   
### 메시 토폴로지
<img src="https://images.edrawsoft.com/kr/articles/edrawmax/topology/mesh-topology.png" width="700" height="300">

-  네트워크상의 모든 노드가 상호 연결된 구조를 가진 토폴로지를 말한다. 
-  완전 메시 형과 부분 메시 형이 있다. 
-  전송 방식은 라우팅과 플러딩의 두 가지 방법이 있다.

#### 장점

1. 안정성과 보안성이 뛰어나다. 
2. 단일 장치의 고장이 네트워크 전체에 영향을 주지 않는다는 점이 있다.
   
#### 단점

1. 설치가 노동 집약적이고 시간이 많이 소요된다.
2. 노드 간의 연결에 필요한 케이블 비용이 많이 들고, 이에 따라 설치 비용이 높아진다.
